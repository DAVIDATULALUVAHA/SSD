Group 2 Team Report

A. Design Proposal Report

Domain

The Dutch Police Internet Forensics (Government of the Netherlands, n.d.)

System Design

The proposed solution will use symmetric-key cryptography to encrypt and decrypt short confidential messages using the same key. The messages form part of the evidence collected while conducting sensitive internet forensic investigation. According to Winderickx et al (2021) symmetric-key cryptography (Figure 1 in Appendix) solves the problem of privacy, unlinkability, shared validation, seamless secret transmission and distributed validation preventing the authentication server from being a single point of attack.

Overview of the Proposed Solution

Figure 2 in the Appendix, shows the UML diagram of the proposed system. The users will be able to encrypt a short message using a key then send the same to the recipient who will decrypt the same using the same key sent to them. The encrypted message will be able to be stored in MySQL database as well as on their desktop as a binary file where the users can be able to retrieve them.

Business and Technical Challenges

  1. The organization is having a challenge of managing and storing important short messages that form part of evidence collected online.
  2. The organisation needs to ensure that they are adhering to privacy and security regulation when dealing with confidential data.
  3. The organisation should ensure that the system is protected against unauthorised access or hacking, malware, and denial of service attacks (Digitpol, 2021).

Approach

The design will be implemented using the Client-Server Architecture as shown in Figure 3 in the Appendix, the client could be a desktop computer, a laptop or Ipad used by users in the organization while the server will be MySQL database that will manage users and store the encrypted data. The cryptography software will also reside of the server.

Theories and Practices used on the Project

According to Ayash (2010) the word "Cryptography” comes from Greek term kruptos, denoting "hidden". The secret of hiding data is to develop a hiding (encryption) system that is very challenging to locate the original data without using the decryption key. Cryptography is the science that utilizes arithmetic to encode and decode data. Sensitive data is encoded and sent over insecure networks to prevent unintended recipient form reading it.

System Requirements

  1. The anonymity and unlinkability of the users of the system must be guaranteed through symmetric-key cryptography (Winderickx et al, 2021).
  2. Perfect forward secrecy should be maintained to prevent adversaries from obtaining encrypted information (Winderickx et al, 2021) .
  3. The system should protect the data from potential attacks and vulnerabilities.
  4. The system must store encrypted information in a database and store a copy of the same in c drive in binary files.
  5. User training must be conducted to ensure that users understand the system and work with it us intended.

Assumptions

  1. The system is assumed to be monitored by the information technology department of the Dutch Police Internet Forensics hence hackers should not be able to penetrate the database system.
  2. End users will communicate securely using the new system and abandon legacy systems of communication.
  3. End users will accept the new system.

Symmetric Encryption

The symmetric cryptography prototype has been inspired from King (2018) and modified to be able to encrypt and decrypt data while the output is stored in MySQL database. Every time a user creates a new message, a key is auto generated and used to encrypt the message. The encoded message and the key are then stored in the database. The key is then used on a later date and time to decrypt the message by the intended recipient. Figure 7 shows symmetric encryption source codes used to encrypt the messages. To encrypt a message, On the menu click on Encrypt to encrypt the message and the form shown in Figure 8 will open. Enter the message and the recipient’s name then click save.

Figure 9 shows how the records look like in MySQL WorkBench. The keys and messages are seen as blobs. This helps to secure the data from unauthorized users. A point to mention is that in a real life application, it would be a better option for the encryption keys to be stored in a separate server. The datatypes can also be renamed to conceal what they store.
